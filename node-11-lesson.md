Задание: REST API «Кинотека» на Express
Цель: научиться создавать сервер на Express, работать с переменными окружения через dotenv и правильно обрабатывать POST-запросы (тело запроса, валидация, статус-коды).
Стек: Node.js, Express, dotenv. База данных не нужна — храним всё в обычном массиве в памяти.
Подготовка

Инициализировать проект (npm init -y), установить express и dotenv.
Создать файл .env с переменными:

PORT — порт сервера
APP_NAME — название приложения

Добавить .env в .gitignore, а рядом положить .env.example.
Сервер должен запускаться на порту из .env, а при старте выводить в консоль: {APP_NAME} запущен на порту {PORT}.

Стартовые данные — массив из 3–4 фильмов вида:

```js
{ id: 1, title: "Интерстеллар", year: 2014, genre: "sci-fi", ratings: [8, 9], reviews: [] }
```

GET-эндпоинты (3 шт.)

GET /api/movies — вернуть все фильмы.
GET /api/movies/:id — вернуть один фильм по id. Если не найден — 404 и { "error": "Movie not found" }.
GET /api/movies/search?genre=sci-fi — вернуть фильмы по жанру из query-параметра. Если параметр не передан — 400.

POST-эндпоинты (3 шт.) — основная часть
Не забыть подключить express.json()!

POST /api/movies — добавить фильм.

Тело: { "title": "...", "year": 2020, "genre": "..." }
Валидация: title — непустая строка, year — число от 1895 до текущего года, genre — строка. Если что-то не так — 400 с описанием, какое именно поле невалидно.
id генерируется на сервере, клиент его не присылает.
Успех — 201 и созданный объект целиком.

POST /api/movies/:id/ratings — добавить оценку фильму.

Тело: `{ "value": 8 }`, число от 1 до 10, иначе 400.
Фильм не найден — 404.
Успех — 201 и объект `{ "average": 8.5, "count": 3 }` с пересчитанным средним.

POST /api/movies/:id/reviews — добавить отзыв.

Тело: { "author": "...", "text": "..." }, оба поля обязательны, text — минимум 10 символов.
Успех — 201, вернуть отзыв с добавленным на сервере полем createdAt.
